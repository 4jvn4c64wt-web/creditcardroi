# _archive

Material that's no longer load-bearing for active work, but worth preserving in case it's referenced later.

## What goes here

- Superseded or reversed entries from `decisions/`.
- Outdated sections pulled out of `CLAUDE_ONBOARDING.md`, `WORKFLOWS.md`, or other docs during compaction.
- Old versions of `ARTICLE_TEMPLATE.md` when the template structure changes.
- Article drafts that were abandoned (shipped drafts go to git history via `insights/`).

## What does NOT go here

- Active reference material. If something is still relevant, leave it in its working location.
- Anything from the site repo (`site/creditcardroi/`). That's tracked in git.
- Anything actually deletable. Archive is not a trash bin — it's storage for things that might matter later. If you're certain nobody will ever want this, just delete it.

## Conventions

- Mirror the original path. If you archive `decisions/2026-04-29-plugin-extraction.md`, put it at `_archive/decisions/2026-04-29-plugin-extraction.md`.
- Add a one-line note at the top of the archived file: `*Archived YYYY-MM-DD — [reason].*`
- Don't reorganize the archive. The original path is the index.
